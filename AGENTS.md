# RevLog

Motorsport life organizer — event tracking, garage management, maintenance
logging, run/lap time tracking. Mobile-first, multi-user, full data privacy
between accounts.

## Stack

- Next.js 15 App Router, React 19, TypeScript
- Tailwind CSS + shadcn/ui (Radix), Inter font, Lucide icons
- NextAuth.js v5, Prisma 7 + PostgreSQL (`@prisma/adapter-pg` + `pg` — WASM engine requires adapter)
- Sonner (toasts), Recharts (charts), react-hook-form + zod, date-fns
- React Compiler is ON — never write useMemo/useCallback/React.memo manually

## Development Status

Current status and phase tracking: `DOCS/STATUS.md` — **update this file after completing any phase or workstream.**

## Hard Rules

- Every DB query MUST be scoped by `userId` — no exceptions
- All API routes MUST call `auth()` and check session before anything else
- Return `{ data }` on success, `{ error: string }` on failure from API routes
- Never use `any` — use `unknown` and narrow it
- Never use deprecated shadcn `toast`/`useToast` — use `sonner`
- Never put DB url in `schema.prisma` — it lives in `prisma.config.ts` (Prisma 7)
- Never expose API keys to the client — Maps/Calendar calls go server-side only
- Mobile-first always. Dark mode default.
- Do not use Supabase client libraries — DB is plain Postgres. Docker in dev, Dokploy VPS in prod
- After completing a phase or workstream, update `DOCS/STATUS.md` — mark items done, record packages added, note any decisions made or open questions resolved
- Use `proxy.ts` not `middleware.ts` — middleware is deprecated in Next.js 16
- Use `useActionState` from `react` — never `useFormState` from `react-dom` (deprecated)
- Google Maps and Calendar are env-gated — check for credentials before calling, fail silently with a no-op if missing
- After completing a major feature or workstream, commit and push to git
- Commit message format: conventional commits — `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`
- Example: `feat: add modifications log with category grouping and cost totals`
- Never include "Co-authored by", "Generated by", "AI-assisted" or any model attribution in commit messages or code comments

## Domain Conventions

- Event types: Autocross, RallyCross, HPDE, Track Day, Time Attack, Drag,
  Test & Tune, Practice Session, Hill Climb, Endurance, Drift, Other
- Registration statuses: Interested, Registered, Waitlisted, Completed, Skipped
- Maintenance alerts: Upcoming (30 days/500 mi), Due (7 days/100 mi), Overdue
- Adjusted run time = raw time + (cones × penalty seconds), default 2s/cone
- Calendar sync triggers on status → Registered, removes on Skipped/deleted

## Testing Strategy (add to CLAUDE.md)

- Write unit tests for all pure utility functions in `src/lib/`
  (time calculations, alert thresholds, penalty math, formatters)
- Write zod schema tests for all form validation schemas
- Do NOT attempt integration or e2e tests until Phase 4
- When a bug is fixed, add a regression test for that specific case
- Test files live next to the source: `formatLapTime.test.ts` beside `formatLapTime.ts`
- Use Vitest (not Jest — better Next.js compatibility)
